# Contributing to RoleMirror

Thank you for your interest in contributing to RoleMirror! We welcome contributions from the community to help improve this project.

## Code of Conduct

Please be respectful and considerate of others when contributing to this project.

## How to Contribute

### 1. Reporting Bugs

If you find a bug, please open an issue on GitHub describing the problem, including:
- Steps to reproduce
- Expected behavior
- Actual behavior
- Environment details (OS, Python version, etc.)

### 2. Suggesting Enhancements

We welcome ideas for new features or improvements. Please open an issue to discuss your suggestion before implementing it.

### 3. Pull Requests

1. Fork the repository.
2. Create a new branch for your feature or fix (`git checkout -b feature/amazing-feature`).
3. Make your changes.
4. Run tests to ensure your changes don't break existing functionality.
5. Commit your changes (`git commit -m 'Add some amazing feature'`).
6. Push to the branch (`git push origin feature/amazing-feature`).
7. Open a Pull Request.

## Development Setup

1. Clone the repository:
   ```bash
   git clone https://github.com/yourusername/RoleMirror.git
   cd RoleMirror
   ```

2. Install dependencies:
   ```bash
   pip install -e .[dev]
   ```

3. Run tests:
   ```bash
   pytest
   ```

## Coding Standards

- We use `black` for code formatting.
- We use `isort` for import sorting.
- Please ensure your code follows these standards before submitting a PR.
